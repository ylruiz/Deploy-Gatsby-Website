Maybe when some visitors look at a website, they see information, pictures, videos. They may also notice the quality of the displayed content, and the level of complexity when they navigate on it. However, if the users are web designers or developers, they see more than that. They probably perceive the code-behind as well as the different areas of expertise that when into building it. That is because various disciplines contribute to the website's creation, but all of them have just one focus: try to make the site friendly use and attractive for the users.

It is not a secret that as a web designer or developer you can work alone, or form part of a small or big team. This means that you may handle many web-related disciplines, or specialize in one or a small number of them.

Whatever the case is, you difinitely should know the following aspects:

  • What roles contribute to the website's creation?
  • What are their respective average salaries?
  • What skills should have every web designer or developer?
  • What are the typical tools of the trade?

What roles contribute to the website's creation?

Multiple disciplines intervene in the process of building a website. Those disciplines include roles related to content, design and code - which may have different responsibilities in the creation of a website.

Let's take a look at the following table, which shows the diverse roles by groups as well as their responsabilities.

1) Content

RoleResponsability
Information Architect (IA)An Information Architect organizes the content in a logical way and makes it easy to find. They may take care for search functionality, site diagrams, and the organization of the content and the data on the server.
Content StrategistA Content Stategist makes sure that every bit of text on a website supports the brand identity and marketing goals of a organization. They may also be responsible for data modeling, content management, and how the organization's voice is represented on social media.

2) Design

RoleResponsability
User Experience (UX)A User Experience designer ensures the entire experience with the website is favorable. They should have a solid understanding of the users and their needs besed on observations and interviews. Their responsabilities also include visual design, the user interface, the quality and message of the content, and the overall site performance.
Interaction Design (IxD)A Interaction Designer has as goal to make the website as easy, efficient, and delightful to use as possible.
User Interface (UI))A User Interface designer focuses on the functional organization and the different elements of the website that are used by the users to navegate through the website's pages or accomplish tasks.

3) Code

RoleResponsability
Frontend developmentFrontend developers focus on the client side, which refers to any aspect of the design proceses that apears in or relates directly to the browser. Frontend development includes the following technologies: HyperText Markup Language (HTML), Cascading Style Sheets (CSS), JavaScript (JS) and DOM scripting including AJAX and JS-based frameworks.
Backend developmentBackend developers focus on the server side, including the applications and databases that run on it. They should domain the following technologies: Server Software, Web application languages, and database software.
Full-Stack developersFull-Stack developers are individuals who are fluent in both frontend and backend.

There are other roles related to web design, such as:

  • Product manager

    A product manager guides the design and production of a website or application in a way that meets business goals. They develop the overall strategy for the website from a marketing perspective, including how and when it gets released.

  • Project manager

    A project manager coordinates the team members managing thinks like timelines, development approaches, deliverables, an so on. They work with product managers and other product owners to ensure that the project gets done on time and on budget.

  • SEO specialist

    A Search Engine Optimization (SEO) specialist focuses on improving the website structure and code to increase the chances it will be highly ranked in a search results.

What skills should have every web designer or developer?

Some non-technical skills that are just as critical as the technicals ones for your success.

  • Excellent skill comunications

    In your work, you will need to communicate with your clients, team members, and superiors. You should communicate with them in a clear, proactive and straighforward way. This also requiere to be a good listener as well as understand issues that are being discusses asking for clarifications if you don't.

  • Flexibility

    When you are working as web designer or developer, you should be aware that not only web technologies change quickly. There can be changes in the projects that you are doing, in your team, etc. So, it is important to be as adaptable as possible to changes due to that is key to survival in any profession.

  • Critical thinking and good judgment

    Problem-solving is central to all web designer or developer, so you should be able to think in a critical way in order to come up with solutions and always employ basic common sense.

  • A good attitude

    When you create a website, you will normally be part of a team, even if you are working freelancer. This means you will need to approach your work being positive and friendly with your clients and your team members.

What are the typical tools of the trade?

It should not be a surprise that web designers need some amount of gear, both hardware and software. So, the next table provides an overview of the typical tools of the trade:

ToolsDescription
EquipmentFor a confortable web development environment, you should have the following devices: a solid computer, a large monitor, a second computer for testing can be also useful, mobiles devices for testing, and a scanner and/or camara if you want to do your own images and textures.
Coding toolsThere are code editors that make the work easier. Some of the most popular are Microsoft Visual Studio, Atom, Sublime Text, Brackets, etc.
UI and layout toolsInterface design tools make it easy to design multiple layouts as well as export images and code for use in production. Some of them also allow basic interactivity such as clicks and swipes, making possible to share your markup online and use it for basic interface testing. Here are some of the most popular options at the time of this writing: Sketch which is only disponible for Mac, Affinity Designer, Adobe XD, Figma, In Vision, and UXPin.
Web graphic creation toolsEven though it is possible to create the images that you need using one interface design tool, there are also programs that focus only on the creation of images. For example, Adobe Creative Cloud, for this one you need to pay a monthly subscription fee. However, there are other lower-cost alternatives that you can try such as GIMP , which is free and open source.
BrowsersA website can look different from browser to browser, so there are some browsers that disigners and developers keep around for testing: Chrome, Firefox, MS Edge (Windows only), Internet Explore 9-11 (Windows only), Safari (Mac only), and Opera.

Web designers and developers usually collaborate with a team or with their clients, depending on their responsabilities. For that reason, it is important to note that they also utilize a certain set of tools in order to make easier the communication between members of the team. For instance, they normally use:

  • Version Control Platforms: GIT, GitHub
  • Project management software: Teamwork, Trello
  • Team chat software: Skype
