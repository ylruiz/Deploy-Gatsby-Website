What is a website?

First of all, let's talk about the internet versus the web. These are two words used interchangeably by many people but they don't mean the same.

The internet is an international network of connected computers that uses the Internet Protocol Suite (TCP/IP) to communicate between networks and devices. Its purposes is to share information between devices (computers, laptops, phones, watches, etc), using standarized methods for transfering data or documents over a network. This can be done through many ways such as email (POP3/IMAP/SMTP), file transfer (FTP), secure shell (SSH), and many others.

On the other hand, the web which it's also know as the World Wide Web is a subset of the internet. It is just one of the many ways in which information can be shared or transferred over the networked computers. The web uses a protocol called HTTP (HyperText Transfer Protocol).

Secondly, the computers that make up the internet are known as servers. However, it is important to note that the server is not the computer itself. Actually, the server is the software that allows the computer to communicate with other devices. Afer that clarification, just rest to say that the web servers or HTTP servers are those computers that form part of the web because they run special web server software that allows them to handle Hypertext Transfer Protocol transactions.

Finally, a website is a collection of publicly accessible, related web pages that share a common domain name in the the web. Now that you know what a website is, you also should understand how the web works which is illustrated in the following image.

With this in mind, it is moment to focus on what happen in each step:

  1. As you probably should know the first thing that you need to do in order to connect to a website is to type the URL of that site in the browser of your preference.

    a) What is an URL?

    Every page and resource on the web has an unique web page address or URL, which stands by Uniform Resources Locator. A complete URL has three components: the protocol, the site name, and the absolute path to the document or resources.

    • http:// indicates to the browser the protocol that will be used. HTTP stands by HyperText Transfer Protocol, so this just says to the browser that it should get into "web mode". You can also see that an URL can begin with https://, the extra "S" in HTTPS stands for "Secure" which means that HTTPS it's the secure version of the standard HTTP.

    • The next part of an URL is the name of the site which is formed by the hostname and the domain name. In this example "www." is the hostname which has become a covention but is not a rule, and it can be omitted.

    • The last part of an URL is the absolute path which is formed by the directory path and the specific HTML document you are looking for. In this example, the directory path is "/2020/samples/" and the HTML document is "first.html".

    b) What is a browser?

    According to wikipedia, a browser is also known as web browser. Browsers are nothing else that special computer programs for displaying websites on the web or for documents and data in general.

    In the moment that you finish to type the URL and press ENTER, the browser sends a request to the Domain Name System (DNS) asking where the URL is.

  2. After that, the DNS responses to the browser with the information of the Iternet Protocol Address(IP Adress) corresponded to the domain name of the URL. The IP Address of every computer and diveces connected to internet has a unique IP address.

    For example, let's suppose that for "example.com" the respective IP Address is 158.241.246.14. This means that the response of the DNS will be something like "example.com is at 158.241.246.14".

  3. When the browser receives the response of the DNS, it sends a new request asking where the URL is. However, in this occasion, it requeses to the Web Server which IP Adress matches with the one sended by the DNS.

    For instance, the Web Server that hosts "example.com" has the IP Address "158.241.246.14". This means that the browser will look for "example.com" at "158.241.246.14".

    Then the Web Server sends the browser the page that the user is looking for, allowing the user to see it if the page is found.

Types of websites

In general terms, the types of websites can be divided into two groups, as you can see in the following table.

Static SitesDynamic Sites
Static sites consists of HTML files with fixed content that displays the same information to every visitor. In other words, each page you see in the browser is a view of a single HTML file on the server.Dynamic sites are generated with backend programming. this means that each page is generated by the application on the fly. Some examples of static websites are those that do NOT allow to store users, to give a comment, to post a blog, or to interact with the site.
